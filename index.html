The tinyMem Story

Originally conceived as a targeted solution to solve memory drift and hallucinations in small, locally-hosted LLMs (7B-13B), tinyMem has evolved into a comprehensive control protocol for repository-aware AI agents.

We discovered that giving an AI "perfect recall" was only half the battle. The other half was enforcement—ensuring that agents actually check their memory before acting and validate every claim against codebase evidence.

# 1. Install tinyMem (macOS/Linux)
os="$(uname -s | tr '[:upper:]' '[:lower:]')"
arch="$(uname -m)"; [ "$arch" = "x86_64" ] && arch="amd64"; [ "$arch" = "aarch64" ] && arch="arm64"
curl -L "https://github.com/daverage/tinyMem/releases/latest/download/tinymem-${os}-${arch}" -o tinymem
chmod +x tinymem && sudo mv tinymem /usr/local/bin/

# 2. Initialize in your project
tinymem health

# 3. Start the "Brain" (MCP mode for IDEs)
tinymem mcp

Evidence-Based Truth

TinyMem uses Typed memories (fact, decision, constraint). It doesn't just guess; it validates claims against your codebase using evidence like file existence, grep hits, and test results.

The Ralph Loop

An autonomous repair engine that iterates until your code works. It runs tests, recalls failure patterns, and applies fixes until the evidence predicates pass—all under your control.

tinyTasks

A built-in, file-authoritative task manager. Agents update tinyTasks.md so progress stays explicit, reviewable, and local to the repo.

The Control Protocol

tinyMem enforces a strict operational sequence that eliminates AI hallucinations and ensures every decision is grounded in codebase reality.

1
Memory Recall
Agent retrieves relevant project context before any reasoning begins.
2
Truth Check
Agent acknowledges state and verifies existing facts against evidence.
3
Task Lock
Multi-step operations are locked to tinyTasks.md for persistent state.
4
Verified Action
Execution occurs, followed by durable memory writeback of results.

MCP Server

Works with MCP-capable clients (including Claude Desktop, Cursor, and VS Code via MCP adapters). tinyMem provides tools like memory_query and memory_ralph through the Model Context Protocol.

HTTP Proxy

OpenAI-compatible HTTP proxy for many SDKs. Point your base URL to localhost:8080 and tinyMem can inject context into requests when configured.

Agent Contracts

Optional directives for common models like Claude, Gemini, and Qwen. Use the provided prompts as a starting point and tailor them to your workflow.

Advanced Reliability

Built for production-grade AI workflows where "maybe" isn't good enough.

  • Chain-of-Verification (CoVe) LLM-based quality filter that reduces hallucinations by verifying memories before they are stored or recalled.
  • Recall Tiers Prioritizes information by importance: Facts & Constraints > Decisions > Claims & Notes.
  • Evidence Predicates Truth validation via file_exists, grep_hit, cmd_exit0, and test_pass.
  • Token Economics Optimized for efficiency, reducing the "Read All Files" waste by providing surgical context snippets.
